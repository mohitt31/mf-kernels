# Assembly Verifications

This directory describes the compiler behaviors referenced in the main `README.md`. The actual assembly output is environment-specific and is not committed, but the findings below were verified on the target architecture.

## Assembly Output: `pitfall` vs `naive`

Generated by running:
```bash
g++ -S -O3 -march=native -I include -x c++ ...
```

**Compiler:** gcc 13.3.0
**Architecture:** x86-64, AVX2 (AMD EPYC)

The resulting assembly confirms that the "helpful" `pitfall` variant causes `gcc 13.3.0` to lower the broadcasts using `vpermpd` (8 occurrences in the loop), whereas the `naive` variant compiles to a clean `vbroadcastsd` (1 occurrence). This difference is the source of the 3–4× performance degradation.
